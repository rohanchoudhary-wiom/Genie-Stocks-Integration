Genie Pipeline · Architecture v2

RF & HGB Concordance Sweep
Model Training Pipeline

A randomised hyperparameter search over Random Forest and HistGradientBoosting regressors, swept across four concordance label-cleaning thresholds. Targets broadband installation prediction via installed_decision.

composite = 2 × AUCtest + gappp · θ ∈ {0.75, 0.80, 0.85, 0.90}
§0 — End-to-End Overview
Pipeline Macro Flow
The upstream Genie train period builds BallTree references and computes spatial features that are already baked into test_scored.h5. This notebook takes that pre-scored test data, merges ops features, applies concordance cleaning, then sweeps 960 model configurations.
§1 — Data Roles
What Train & Test Data Are Used For
The pipeline uses three distinct data sources with very different roles. The upstream train-period data is never used for RF/HGB model training — it only supplies the BallTree reference labels and the pre-computed spatial features that are already populated into the test-scored file.

Key insight: train_data.h5 is never used to train RF/HGB. It serves two upstream purposes: (1) computing spatial features pre-populated into test_scored.h5, and (2) providing BallTree reference labels for concordance gating. All model training happens on a 70% split of test_scored.h5.
Concordance flow: BallTree (from train_data.h5 hard labels) is queried for each ambiguous lead in test_scored.h5. If ≥3 train neighbours within 100m show ≥θ agreement, the lead is promoted. This is applied only to the 70% model-train fold — the 30% eval-test fold stays untouched.

§4 — Label Cleaning
Concordance-Gated Label Filtering
Ambiguous leads (INDETERMINATE/HANGING) from test_scored.h5 are rescued or discarded based on spatial concordance with train_data.h5 hard labels.
§5 — Train / Test Protocol
Stratified Split on test_scored.h5
The full modelling dataset is split 70/30 stratified on target first. Concordance filtering is applied only to the 70% model-train fold. The 30% eval-test fold stays untouched.

§7 — Concordance Sweep Loop
4 × 240 = 960 Model Evaluations
The outer loop iterates thresholds. For each, the model-train set is re-filtered, then RF and HGB sweeps execute. Each model is evaluated on the same untouched eval-test fold and scored with composite = 200 × AUC_test + gap_pp.
