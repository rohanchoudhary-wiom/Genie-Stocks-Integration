Section 0

Target variable & KPI

The pipeline predicts a single binary outcome and optimises a composite metric that balances ranking quality with decile separation.

f(x)

Composite KPI

composite = AUC_test × 200 + decile_gap_pp
Higher is better. AUC measures rank-ordering quality; decile_gap_pp = D10 SE − D1 SE in percentage points.
MetricDescriptionRange
installed_decision Binary target. 1 = lead was installed, 0 = not installed. {0, 1}
AUC Area Under ROC Curve — rank-ordering quality of predicted scores. 0.5 – 1.0 (scaled ×200)
decile_gap_pp D10 install rate − D1 install rate, in percentage points. Measures separation between best and worst deciles. 0 – ~60 pp
composite AUC×200 + gap_pp. Single number to maximise during sweep. Typical range: 50–160. ~50 – 160
mono Boolean — are decile SE values monotonically increasing? Indicates clean calibration. True / False

Section 2

SQL queries (Snowflake)

Two Snowflake queries power cross-partner spatial density features. Both use BallTree on the result set for efficient radius counting per lead. Results are cached to HDF5 on first run.

Active user base density
Cross-partner — counts all paying Wiom customers nearby, not just same-partner. Cached to active_user_base.h5.
t_router_user_mapping → latest_plan CTE → t_wg_customer (dedup) → t_active_base (lat/lng) → BallTree
WITH latest_plan AS ( SELECT mobile, router_nas_id, DATEADD(minute,330,otp_issued_time) AS plan_start_ist, DATEADD(minute,330,otp_expiry_time) AS plan_expiry_ist, ROW_NUMBER() OVER ( PARTITION BY router_nas_id ORDER BY otp_expiry_time DESC, created_on DESC ) AS rn FROM t_router_user_mapping WHERE auth_state=1 AND device_limit='10' AND otp NOT IN ('FREE','PAY_ONLINE','CASH','ROAM') AND mobile>'5999999999' AND charges>=19 AND CAST(DATEADD(minute,330,created_on) AS DATE) BETWEEN DATEADD(day,-400,'END_DT') AND 'END_DT' ), twc_dedup AS ( SELECT idmaker(shard,0,nasid) AS lng_nasid, idmaker(shard,4,account_id) AS lng_cust_id, ROW_NUMBER() OVER ( PARTITION BY idmaker(shard,0,nasid) ORDER BY added_time DESC) AS rn FROM t_wg_customer ) SELECT lp.router_nas_id, lp.plan_start_ist, lp.plan_expiry_ist, ab.lat AS latitude, ab.lng AS longitude FROM latest_plan lp JOIN twc_dedup twc ON lp.router_nas_id=twc.lng_nasid AND twc.rn=1 JOIN t_active_base ab ON twc.lng_cust_id=ab.long_customer_id WHERE lp.rn=1 AND ab.lat IS NOT NULL
✓

Active at t0 = plan_start_ist ≤ t0 AND plan_expiry_ist ≥ t0. Cross-partner — a lead in Partner A's territory still counts nearby users from Partner B.

Data consumption density
Cross-partner. 30-day trailing GB per NAS from data_usage_okr. Cached to data_consumption.h5.
data_usage_okr (30d) → SUM(total_data_used) → t_wg_customer (dedup) → t_active_base (lat/lng) → BallTree
WITH twc_dedup AS ( SELECT idmaker(shard,0,nasid) AS lng_nasid, idmaker(shard,4,account_id) AS lng_cust_id, ROW_NUMBER() OVER ( PARTITION BY idmaker(shard,0,nasid) ORDER BY added_time DESC) AS rn FROM t_wg_customer ), usage_agg AS ( SELECT nasid, SUM(total_data_used) AS total_gb, COUNT(DISTINCT dt) AS active_days FROM data_usage_okr WHERE dt BETWEEN DATEADD(day,-30,'END_DT') AND 'END_DT' GROUP BY nasid HAVING SUM(total_data_used) > 0 ) SELECT ua.nasid, ua.total_gb, ua.active_days, ab.lat AS latitude, ab.lng AS longitude FROM usage_agg ua JOIN twc_dedup twc ON ua.nasid=twc.lng_nasid AND twc.rn=1 JOIN t_active_base ab ON twc.lng_cust_id=ab.long_customer_id WHERE ab.lat IS NOT NULL

Three families of partner-derived features are merged onto each lead: geographic proximity to partner territory, operational performance metrics, and nearby active bookings.

Section 4

Spatial & proximity computation

All spatial features use BallTree with haversine metric on (latitude, longitude) in radians. Earth radius constant = 6,371,000 metres. Rings are mutually exclusive (not cumulative).

BallTree computation pattern

StepOperationDetail
1. Coordinate prep np.radians(df[['latitude','longitude']]) Convert degrees to radians for haversine metric
2. Build tree BallTree(coords_rad, metric='haversine') One tree per partner (bookings, geo) or one global tree (users, data consumption)
3. Nearest query tree.query(pt, k=1) Returns distance in radians → multiply by EARTH_R for metres
4. Radius query tree.query_radius(pt, r=r_m/EARTH_R) Returns all indices within radius. Mutually exclusive rings computed by filtering: (dist > lo_r) & (dist <= hi_r)
5. Temporal filter Active at t0 Only for user/booking features: start ≤ t0 AND (expiry ≥ t0 OR NaT)

Section 6

Pipeline steps

From raw features to trained models. Collinearity analysis removes redundant pairs before the sweep.

Step 1

Feature selection

Build list from registry, check availability in df

Step 2

Correlation analysis

Feature–target r, collinear pairs |r| > 0.7

Step 3

Prep & clean

fillna(-999), cast to float64, drop NaN target

Step 4

Train/test split

70/30 stratified split, seed=42

Step 5

HP sweep

120 random combos × 4 model types

Step 6

Evaluate

AUC, decile gap, composite, monotonicity

Config constants

ParameterValuePurpose
N_RANDOM_SAMPLES120Random combos per model type in sweep
TEST_SIZE0.3030% held out for test
RANDOM_STATE42Global seed for reproducibility
FILL_VALUE-999Missing value fill for tree models
COLLINEAR_THRESHOLD0.7Pearson r above which pairs are flagged

Evaluation: Each model is scored on AUC (train + test), decile table (10 quantile bins), D1/D10 gap in pp, monotonicity check, and the composite KPI.
