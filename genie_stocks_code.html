B · SPATIAL
Gaussian KDE Field
kernel(s) = exp( −0.5 × d²/h² ) time(s) = exp( −λ × age_days ) λ=0.005 field(t) = Σ(kernel × w × time) / Σ(kernel)
Called twice: per-hex field (predicted_field_hex) and all-hex combined field. h = adaptive bandwidth from 3-NN (20–150m).
B · SPATIAL
Asymmetric SE Shrinkage
ratio = max(0.45, total / (total + 4)) se ≥ 0: se_shrunk = se × ratio dampen positive se < 0: se_shrunk = se / ratio amplify negative
Applied per hex in get_parent_hexagon() before evidence-weighted consensus across all covering hexes.

hop_k_wmean = Σ(w × SE) / Σ(w) w = 1/(hop×dist) gradient = hop1 − hop3 local pocket signal confirmed = hop1 × hop3 both rings good isolation = count1 / count3 fringe vs core

B · Spatial — Training & Test Dataget_data.py → get_train_data(), get_test_data()

get_train_data() / get_test_data() Fetches historical decisions for map training and test lead scoring. Same query, different date windows.
t_node_decisions_active → partner_id, mobile, lat/lng, installed_decision, final_decision feeds → IDW field weights, hex generation, scoring
Snowflake SQL t_node_decisions_active 
SELECT
    partner_id, mobile, first_notified_time,
    CASE WHEN lco_account_installed = partner_id
         THEN 1 ELSE 0 END AS installed_decision,
    latitude, longitude, installed_time,
    CASE WHEN lco_account_installed = partner_id
         THEN 'INSTALLED'
         ELSE first_event END AS final_decision,
    active_base, partner_tenure,
    first_event_time, last_event, last_event_time, first_event
FROM t_node_decisions_active
WHERE first_event NOT IN ('', 'None')
  AND CAST(first_notified_time AS DATE) BETWEEN '{start_dt}' AND '{end_dt}'
Raw → process_dataframe() → standardise_decisions() → INSTALLED / DECLINED / INDETERMINATE / HANGING
Standardisation logic: Computes p40 of time_to_decide for INTERESTED→DECLINED. Fast declines (≤p40) → DECLINED. Slow declines (>p40) → INDETERMINATE. Still pending → HANGING. Then IDW-interpolated field_weight for ambiguous points.
get_g1_distance() G1 serviceability check logs — min distance from lead to nearest partner coordinate. Used for distance-bucket analysis and decline simulation.
t_serviceability_logs → mobile, min_dist, dist_rng, lat/lng, zone_alias, serviceable feeds → test merge (min_dist_bucket) + step 3 simulation
Snowflake SQL t_serviceability_logs 
SELECT mobile, created_at AS decision_time, min_dist,
  CASE
    WHEN min_dist <= 5  THEN 'A. 0-5m'
    WHEN min_dist <= 10 THEN 'B. 5-10m'
    -- ... bins in 5m increments up to 100m ...
    ELSE 'U. >100m'
  END AS dist_rng,
  latitude, longitude, g1_is_bdo_lead, zone_alias, serviceable
FROM (
  SELECT *, ROW_NUMBER() OVER(PARTITION BY mobile ORDER BY created_at DESC) AS row_cnt
  FROM (
    SELECT *,
      PARSE_JSON(response):extra_data.nearest_coordinate.distance AS min_dist,
      PARSE_JSON(response):"serviceable"::BOOLEAN AS serviceable,
      PARSE_JSON(response):extra_data.nearest_coordinate.zone_alias::STRING AS zone_alias,
      PARSE_JSON(response):genie_downstream_context.g1_is_bdo_lead::BOOLEAN AS g1_is_bdo_lead,
      SPLIT(PARSE_JSON(response):genie_downstream_context.g1_lead_coords, ',')[0]::FLOAT AS latitude,
      SPLIT(PARSE_JSON(response):genie_downstream_context.g1_lead_coords, ',')[1]::FLOAT AS longitude
    FROM prod_db.mysql_rds_genie_genie1.t_serviceability_logs
    WHERE CAST(created_at AS DATE) BETWEEN '{start_dt}' AND '{end_dt}'
  )
) WHERE row_cnt = 1  -- latest log per mobile
Parses JSON response payload to extract distance, coordinates, and serviceability. ROW_NUMBER deduplicates to latest log per mobile.

B · Operational — Capacity, Tickets, Reliabilityget_ops_data.py → build_partner_ops_vector()

get_pending_leads_per_partner() Lead lifecycle reconstruction. Joins interest → decline/install/cancel events. Rows with ALL terminal events NULL = pending (feeds E exposure).
task_logs booking_logs → mobile, partner_id, interest_time, decline_time, install_time, cancel_time feeds → compute_lead_capacity() → nmbr_active_leads, queue_velocity
Snowflake SQL — 4 CTEs joined task_logs + booking_logs 
WITH interest AS (
  SELECT mobile,
    TRY_PARSE_JSON(data):partner_id::STRING AS partner_id,
    added_time AS interest_time,
    ROW_NUMBER() OVER(PARTITION BY mobile, partner_id ORDER BY added_time DESC) AS rn
  FROM prod_db.public.task_logs
  WHERE event_name = 'NOTIF_SENT'
    AND CAST(added_time AS DATE) BETWEEN '{start}' AND '{end}'
),
decline AS (
  SELECT mobile, partner_id,
    added_time AS decline_time,
    ROW_NUMBER() OVER(PARTITION BY mobile, partner_id ORDER BY added_time ASC) AS rn
  FROM prod_db.public.task_logs
  WHERE event_name IN ('DECLINED', 'NOT_INTERESTED')
),
install AS (
  SELECT mobile,
    TRY_PARSE_JSON(data):lco_account_id::STRING AS partner_id,
    added_time AS install_time, ROW_NUMBER() ... AS rn
  FROM prod_db.public.booking_logs
  WHERE event_name = 'lead_state_changed'
    AND TRY_PARSE_JSON(data):state::STRING = 'installed'
),
cancel AS (
  -- same pattern, state IN ('cancelled','unservisable','refunded')
)
SELECT i.mobile, i.partner_id, i.interest_time,
  d.decline_time, inst.install_time, c.cancel_time,
  COALESCE(d.decline_time, inst.install_time, c.cancel_time) AS decision_time
FROM interest i
LEFT JOIN decline  d    ON i.mobile=d.mobile AND i.partner_id=d.partner_id AND d.rn=1
LEFT JOIN install  inst ON ... AND inst.rn=1
LEFT JOIN cancel   c    ON ... AND c.rn=1
WHERE i.rn = 1
Raw rows → compute_lead_capacity(eval_time) → pending = decision_time IS NULL → long_held = pending AND open > 24h → queue_velocity = resolved / total
get_ticket_tasks() Service / cash_collection / router_pickup tickets. Uses LEAD() window to compute how long each partner held each task.
ticketvanilla_audit → partner_id, ticket_type, tat_open_minutes, is_resolved, is_still_open feeds → compute_ticket_capacity() → active_tickets, median_tat
Snowflake SQL ticketvanilla_audit 
WITH assign_events AS (
  SELECT
    account_id AS long_customer_id,
    task_id, added_time AS assign_time,
    assigned_account_id AS partner_id,
    type AS ticket_type,
    LEAD(added_time) OVER(
      PARTITION BY task_id ORDER BY added_time
    ) AS next_event_time,
    status
  FROM prod_db.public.ticketvanilla_audit
  WHERE event_name IN ('ticket_assigned', 'ticket_reopened', 'task_created')
    AND LOWER(assigned_to) = 'partner'
    AND LOWER(type) IN ('service', 'cash_collection', 'router_pickup')
)
SELECT
  long_customer_id, task_id, partner_id, ticket_type,
  assign_time, next_event_time,
  DATEDIFF(MINUTE, assign_time,
    COALESCE(next_event_time, CURRENT_TIMESTAMP)) AS tat_open_minutes,
  CASE WHEN LOWER(status) = 'resolved' THEN 1 ELSE 0 END AS is_resolved,
  CASE WHEN next_event_time IS NULL THEN 1 ELSE 0 END AS is_still_open
FROM assign_events
Raw → compute_ticket_capacity() → per-partner: active_tickets, median_tat_min, max_tat_min, ticket_resolution_rate
get_slot_confirmation() Measures partner reliability: gap between slot confirmation and plan creation. Close-distance tardiness penalised more heavily.
task_logs (CUSTOMER_SLOT_CONFIRMED) booking_logs (trum_plan_created) → partner_id, slot_time, plan_time, late_arrive_days, nearest_distance feeds → compute_reliability() → late_severity, plan_created_rate
Snowflake SQL task_logs ⋈ booking_logs 
WITH slots AS (
  SELECT mobile,
    TRY_PARSE_JSON(data):partner_id::STRING       AS partner_id,
    TRY_PARSE_JSON(data):slot_selected::TIMESTAMP  AS slot_selected_time,
    TRY_PARSE_JSON(data):nearestDistance::FLOAT   AS nearest_distance,
    added_time AS slot_confirmed_time,
    ROW_NUMBER() OVER(PARTITION BY mobile ORDER BY added_time DESC) AS rn
  FROM prod_db.public.task_logs
  WHERE event_name = 'CUSTOMER_SLOT_CONFIRMED'
    AND added_time >= '{start_dt}' AND added_time <= '{end_dt}'
),
plans AS (
  SELECT mobile, added_time AS plan_created_time,
    ROW_NUMBER() OVER(PARTITION BY mobile ORDER BY added_time ASC) AS rn
  FROM prod_db.public.booking_logs
  WHERE event_name = 'trum_plan_created'
    AND added_time >= '{start_dt}' AND added_time <= '{end_dt}'
)
SELECT s.mobile, s.partner_id, s.slot_selected_time,
  s.slot_confirmed_time, s.nearest_distance,
  p.plan_created_time,
  CASE WHEN p.mobile IS NOT NULL THEN 1 ELSE 0 END AS plan_created_flag,
  DATEDIFF(SECOND, s.slot_confirmed_time,
    COALESCE(p.plan_created_time, CURRENT_TIMESTAMP)) / 86400.0 AS late_arrive_days
FROM slots s
LEFT JOIN plans p ON s.mobile = p.mobile AND p.rn = 1
WHERE s.rn = 1
Raw → compute_reliability() → late_severity (ordinal 0–3) → late_close_penalty = late_days × 1/(distance+1) → planning_strength = plan_rate × count

G + B · Partner Performance & Expected Slotsget_ops_data.py → feeds gatekeeper + ops vector

get_partner_performance() Per-partner SE, decline rate, and median response time. Dual-purpose: feeds G (non-responder gate via decline_rate_30d) and B_ops (se_30d, response metrics).
t_node_decisions_active → total_decisions, installs_30d, declines_30d, median_response_min feeds → G: partner_block_gate (decline_rate ≥ 0.85) feeds → B_ops: se_30d, response metrics feeds → S: decline_spike shock
Snowflake SQL t_node_decisions_active 
SELECT
  partner_id,
  COUNT(*)       AS total_decisions,
  SUM(CASE WHEN partner_id = lco_account_installed
    THEN 1 ELSE 0 END)   AS installs_30d,
  SUM(CASE WHEN first_event = 'DECLINED'
    THEN 1 ELSE 0 END)   AS declines_30d,
  MEDIAN(reaction_time_notif)  AS median_response_min,
  AVG(reaction_time_notif)     AS mean_response_min
FROM t_node_decisions_active
WHERE first_notified_time >= DATEADD(DAY, -30, '{end_dt}')
  AND first_notified_time <= '{end_dt}'
  AND first_event IS NOT NULL
  AND first_event NOT IN ('', 'None')
GROUP BY partner_id
Raw → Python computes: se_30d = installs / total → decline_rate_30d = declines / total
This is the starting frame for the full ops vector merge — all other ops data sources left-join onto this. If this query returns empty, the entire ops vector is empty.
get_expected_daily_slots() Rolling average daily installs per partner over 45-day window. Used by G (capacity_gate), E (exposure ratio), and S (capacity_overload shock).
booking_logs → partner_id, expected_daily_slots, active_days feeds → G: capacity_gate (pending ≥ 3× slots) feeds → E: exposure_ratio = pending / slots
Snowflake SQL booking_logs 
WITH daily AS (
  SELECT
    TRY_PARSE_JSON(data):lco_account_id::STRING AS partner_id,
    CAST(added_time AS DATE) AS dt,
    COUNT(*) AS daily_installs
  FROM prod_db.public.booking_logs
  WHERE event_name = 'lead_state_changed'
    AND TRY_PARSE_JSON(data):state::STRING = 'installed'
    AND added_time >= DATEADD(DAY, -45, '{end_dt}')
    AND added_time <= '{end_dt}'
  GROUP BY 1, 2
)
SELECT
  partner_id,
  AVG(daily_installs)    AS expected_daily_slots,
  COUNT(DISTINCT dt)    AS active_days,
  SUM(daily_installs)    AS total_installs_period
FROM daily
GROUP BY partner_id

S · Shock Ledger — Outage Dataget_ops_data.py → get_active_outages() → compute_shock_flags()

get_active_outages() Active router outages within the recency window (default 7 days). Severity ≥ 3 triggers an outage shock flag.
outage_incidents_aggregated → partner_id, severity, device_count, outage_recency_hours feeds → compute_shock_flags() → has_shock per partner
Snowflake SQL outage_incidents_aggregated 
SELECT
  partner_id,
  severity,
  device_count,
  size_bucket,
  status,
  added_time AS outage_time,
  DATEDIFF(HOUR, added_time, CURRENT_TIMESTAMP) AS outage_recency_hours
FROM prod_db.BUSINESS_EFFICIENCY_ROUTER_OUTAGE_DETECTION_AUDIT_PUBLIC
     .outage_incidents_aggregated
WHERE LOWER(status) = 'active'
  AND added_time >= DATEADD(DAY, -7, '{end_dt}')
  AND added_time <= '{end_dt}'
Raw → compute_shock_flags() assembles 3 shock types:
① outage: max_severity ≥ 3 from this query
② decline_spike: decline_rate_30d ≥ 0.85 (from get_partner_performance)
③ capacity_overload: nmbr_active_leads > 3× expected_daily_slots (from lead_capacity × slots)
Output: has_shock (0/1) + shock_types (pipe-delimited) per partner → merged into ops vector → feeds G shock_gate + B_ops infrastructure_score
